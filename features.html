Personalization

Make it yours

Dev Links adapts to how each developer works. Pin the resources you use most, define personal variables, and switch themes — all saved locally in your browser.

Pin your most-used resources

Click the ★ star icon on any resource to pin it to the dedicated "Pinned" section at the top of the page. Drag to reorder. Your pins persist in localStorage.

  • Star any resource with one click
  • Pinned section always appears at the top
  • Drag-and-drop reordering
  • Saved to localStorage — survives page reload
🖼️
Pinned resources section
screenshots/03-pinned-resources.png

Template variables fill in your URLs

Some URLs contain placeholders like {username} or {app-folder}. Open the ⚙ My Vars panel, enter your values once, and every matching URL auto-fills across the entire app.

  • Variables defined once, applied everywhere
  • Values are private — stored only in your browser
  • Visual indicator on cells that use template variables
  • Clipboard copy includes your resolved values
🖼️

Admin & Data

Keep data fresh and correct

A full admin interface and schema validation ensure the data behind Dev Links stays accurate.

Admin UI — no YAML editing required

Run npm run admin to launch a graphical admin interface. Add, edit, or delete resources, environments, environment sets, and categories without touching a YAML file.

  • Full CRUD for resources, environments, and categories
  • Live schema validation with human-readable error messages
  • Pre-commit hooks catch errors before they ship
  • Consistency checks: unique IDs, known env references
